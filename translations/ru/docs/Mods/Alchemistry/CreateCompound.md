# Создать соединение
Пользовательское поколение алхимической смеси

# Пакет
```zenscript
mods.alchemistry.Util.createCompound
```

## Погрузчик
Это должно быть загружено с `#alchemistry`

## Методы
- **int meta** meta for new Compund
- **Имя строки** Имя для нового компаунда
- **красный красный оттенок составного** RGB
- **int green** compound green shade, RGB
- **int синий** составной синий оттенок, RGB
- **Объект[][]** Каждая скобка содержит другой элемент, за которым следует запятая и количество использованных в вашей Соединенной Республике

## Создание
```zenscript
mods.alchemistry.Util.createCompound(int meta, String name, int red, int green, int blue, Object[][] components);

mods.alchemistry.Util.createCompound(1000, "vibranium sufide", 20, 69, 185,
[["vibranium", 1],
 ["серы", 3]]);
```

## Примечания
- Пожалуйста, используйте имена строчных регистров
- Строка алхимии '#loader ' требуется в верхней части файла. Этот файл может быть использован только для создания элементов и соединений, другие рецепты должны быть помещены в другие файлы.
- Поле компонентов требует каждого элемента как массив ["целлюлоза", 5], указывающий элемент/состав и его количество. Здесь используются строчные аргументы, а не itemstacks для обеспечения того, чтобы используются только элементы/соединения.
- Мета поле обязательно, чтобы убедиться, что у ваших новых соединений есть неизменяемый идентификатор, даже если другие соединения создаются или удаляются. Каждое соединение должно иметь уникальную метаценность. На момент написания мод использует только метазначения от 0 до 35, но я начал бы нумеровать ваш мета на 1000 (как показано выше) или выше, чтобы убедиться, что ваши пользовательские соединения никогда не стыкуются с рецептами алхимии
